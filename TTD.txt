SECTION 1 — PURPOSE AND SCOPE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PURPOSE:
  Build a private, eavesdrop-resistant communication channel between two
  LLM-based agents. The channel must operate with zero natural language on the
  wire at any point. Instead, agents communicate through tensors — numerical
  arrays that encode intent — transformed by a set of cryptographic operations
  before transmission and reduced to a tiny wire payload of approximately
  38 bytes per message.

SCOPE:
  This document covers every module, every data transformation, every LangGraph
  node, and every design decision required to build version 1.0 of the system.
  It does not contain executable code, but is detailed enough that a developer
  can implement each module by reading this document alone.

OUT OF SCOPE:
  - Actual quantum hardware. All "quantum" in this system refers to
    quantum-inspired mathematics running on classical hardware.
  - Network transport layer. The system assumes agents can exchange bytes
    through any channel (TCP socket, HTTP POST, message queue, etc).
  - Multi-hop routing across more than two agents in v1.0.

SECTION 2 — SYSTEM PHILOSOPHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CORE IDEA:
  Standard A2A protocols (Google A2A, IBM ACP, LangGraph messages) transmit
  structured JSON or natural language. Any observer with the protocol spec can
  read the message. This system eliminates that property entirely.

  Agents think in TENSORS. A tensor is a 384-dimensional float32 numpy array
  where each dimension encodes a component of the agent's internal intent
  (task type, entity type, output format, priority, source reference, etc).
  No sentence, no word, no character is ever constructed.

  Before transmission, the tensor is:
    (a) salted with a per-message secret offset
    (b) passed through a non-linear two-layer projection using secret matrices
    (c) quantized to the nearest entry in a secret 4096-entry codebook
    (d) signed with HMAC for integrity

  What travels on the wire is ONE INTEGER (the codebook index, 12 bits),
  ONE 2-CHARACTER STRING (the RCC-8 spatial relation), ONE COUNTER (4 bytes),
  and ONE HMAC (32 bytes). Total: ~38 bytes. An observer sees nothing useful.

GEOSPATIAL LOGIC:
  The RCC-8 (Region Connection Calculus) system is borrowed from formal
  geospatial topology. It defines four spatial relations between two regions:

    EQ  — regions are identical (same concept)
    PO  — regions partially overlap (shared sub-meaning)
    EC  — regions touch at boundary (clean handoff point)
    DC  — regions are disconnected (completely different concepts)

  In this system, "regions" are zones in the 384-dimensional embedding space.
  The relation between the sender's projected tensor and the nearest concept in
  the receiver's ledger determines the COORDINATION STRATEGY — not just the
  message meaning, but what the receiving agent DOES next.

QUANTUM-INSPIRED SUPERPOSITION:
  An intent tensor is not a single point. It is built as a WEIGHTED SUM of
  multiple basis vectors (one per concept dimension), mimicking quantum
  superposition. The receiver "collapses" this superposition by projecting
  the decoded tensor onto its concept basis and taking the argmax. Small
  errors in reconstruction are handled gracefully because collapse is a
  similarity operation, not an equality check.

SECTION 3 — HIGH-LEVEL ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The system has three layers:

LAYER 1 — CRYPTOGRAPHIC FOUNDATION
  Established once per agent pair before any messages are sent.
  Uses ECDH (X25519 elliptic curve) for key exchange.
  Uses HKDF (SHA-256 based) to expand the shared secret into all other
  cryptographic material: projection matrices W1 and W2, the codebook,
  per-message salts, and the HMAC key.
  Nothing in this layer is ever transmitted. It exists only in memory.

LAYER 2 — SEMANTIC ENCODING / DECODING PIPELINE
  The pipeline that transforms intent tensors into wire messages and back.
  This is where the geospatial logic and quantum-inspired math live.
  Implemented as a LangGraph StateGraph with nodes for each pipeline step.
  State is persisted at every step using InMemorySaver.
  Cross-agent shared concepts are stored in InMemoryStore (the ledger).

LAYER 3 — AGENT LOGIC
  The actual work that agents do before encoding (Agent A — sender) and
  after decoding (Agent B — receiver). DeepSeek powers the agents' internal
  reasoning for task construction and result interpretation.
  The DeepSeek LLM is used ONLY for internal reasoning — its outputs are
  NEVER placed on the wire. Only tensors reach the wire.

LANGGRAPH GRAPH TOPOLOGY:

  [KEY_EXCHANGE_NODE]
          |
          v
  [BUILD_INTENT_NODE]  ←── Agent A's internal reasoning (DeepSeek)
          |
          v
  [ENCODE_NODE]        ←── salt + project + quantize → SemanticMessage
          |
          v
  [SIGN_NODE]          ←── HMAC over wire payload
          |
          v
  ─────── WIRE (38 bytes) ─────────────────────────────────────────
          |
          v
  [VERIFY_NODE]        ←── HMAC check + counter freshness check
          |
          v
  [DECODE_NODE]        ←── codebook lookup + desalt + reverse project
          |
          v
  [RCC8_ROUTE_NODE]    ←── reads relation tag → sets strategy
          |
          |──── EQ ──→ [EXACT_DISPATCH_NODE]
          |──── PO ──→ [PARALLEL_SPLIT_NODE]  (uses Send API)
          |──── EC ──→ [HANDOFF_NODE]
          └──── DC ──→ [NEGOTIATE_NODE]       (uses interrupt())
                              |
                              v
                      [COLLAPSE_NODE]        ←── argmax over concept basis
                              |
                              v
                      [EXECUTE_NODE]         ←── Agent B acts on decoded intent
                              |
                              v
                      [RESPOND_NODE]         ←── result tensor built, sent back

SECTION 5 — DEPENDENCIES AND INSTALLATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

REQUIRED PACKAGES:

  langgraph          — The core graph orchestration framework. Install the
                       latest version. Required for StateGraph, InMemorySaver,
                       InMemoryStore, Command, Send, interrupt(), @entrypoint,
                       and @task.

  langchain-openai   — Used to connect to DeepSeek's OpenAI-compatible API
                       endpoint. DeepSeek exposes an endpoint that accepts
                       OpenAI-format requests, so the langchain-openai package
                       works directly with it by changing the base_url and
                       api_key parameters.

  cryptography       — For X25519 ECDH key exchange and HKDF key derivation.
                       This is the Python cryptography library by PyCA.
                       Provides X25519PrivateKey, HKDF, SHA256.

  numpy              — All tensor operations. float32 arrays. Matrix multiply,
                       tanh, arctanh, cosine similarity, QR decomposition.

  lancedb            — Optional but recommended for persistent ledger storage
                       across long sessions. Used instead of InMemoryStore
                       when you want the ledger to survive process restarts.

  streamlit          — Dashboard only. Not required for core system.

  pytest             — Testing framework.

  pydantic           — For validating SemanticMessage wire format and
                       structured outputs from DeepSeek.

INSTALLATION ORDER:
  1. Create a virtual environment with Python 3.11
  2. pip install langgraph
  3. pip install langchain-openai
  4. pip install cryptography
  5. pip install numpy
  6. pip install lancedb  (optional)
  7. pip install streamlit pytest pydantic

ENVIRONMENT VARIABLES REQUIRED:
  DEEPSEEK_API_KEY   — Your DeepSeek API key
  GQESL_LOG_LEVEL    — Set to DEBUG during development, INFO in production
